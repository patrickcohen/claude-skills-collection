Key Information

  • Type: L2 Worker (standalone)
  • Philosophy: Traffic-First, KILL early
  • Input: ideas="a,b,c" OR context="product description"
  • Output: docs/reference/research/[date]-discovery.md
  • Time-box: 15-70 minutes (3-10 ideas)
  • Position: Before ln-210 (Epic Coordinator)

Tools Used

  • WebSearch - Traffic, demand, competition, revenue research
  • AskUserQuestion - Personal interest rating
  • Write - Save output file
  • Bash - Create directories

KILL Thresholds

  • Filter 1 (Traffic): No identifiable channel = KILL
  • Filter 2 (Demand): <1K searches/month = KILL
  • Filter 3 (Competition): Index 5 (>10 competitors) = KILL
  • Filter 4 (Revenue): <$20/user = KILL
  • Filter 5 (Interest): Score 1-2 = KILL
  • Filter 6 (MVP): >8 weeks = KILL

Traffic-First Philosophy

  • Anti-pattern: Idea -> Surveys -> Product -> "where's traffic?"
  • Correct: Traffic -> Niche -> MVP -> Launch under existing demand
  • No surveys: Measure real search demand, not opinions
  • One channel: Focus on single acquisition channel
  • KILL early: Don't waste time scoring bad ideas

Composite Score (Survivors Only)

  • Formula: Demand + (6 - Competition) + Revenue + Interest + MVP
  • Max score: 25
  • GO: Score >= 20
  • CONSIDER: Score 15-19
  • WEAK: Score < 15
